# Member Page (/member) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/member` page (system-wide users list) and a left-nav "Analytics/分析" group entry, where clicking a user expands the user's workspaces + per-workspace role + projects.

**Architecture:** Add two new authenticated API endpoints (`GET /users`, `GET /users/{user_id}/workspaces`) and a new Next.js page `codes/web/app/(app)/member/page.tsx` that loads the list + lazily loads expanded details with simple client-side caching.

**Tech Stack:** FastAPI + SQLAlchemy (codes/core-service), Next.js App Router + client components (codes/web), existing `apiFetch` token auth.

### Task 2: Implement `GET /users` and `GET /users/{id}/workspaces` (GREEN)

**Files:**
- Create: `codes/core-service/app/routes/users.py`
- Create: `codes/core-service/app/schemas/user.py`
- Modify: `codes/core-service/app/main.py`

- [ ] **Step 1: Add schemas**

Create `app/schemas/user.py`:
- `UserOut`: `id, email, display_name, status, workspace_count`
- `UserWorkspaceOut`: `{ workspace, membership, projects }` with nested objects

Keep schema fields aligned with existing ones:
- Workspace fields match `WorkspaceOut` (id, name, description)
- Project fields match `ProjectOut` (id, name, description, archived)
- Membership fields: `id, role, status`

- [ ] **Step 2: Add routes**

Create `app/routes/users.py`:
- Router prefix `/users`, tag `users`
- `GET ""`:
  - Depends `get_current_user` (auth required)
  - Query all users ordered by `created_at desc`
  - For each user, compute `workspace_count` = count of active `WorkspaceMember` rows (status == "active")
  - Return `list[UserOut]`
- `GET "/{user_id}/workspaces"`:
  - Depends `get_current_user`
  - Return memberships for that `user_id` joined with `Workspace`
  - Batch-load `Project` rows by `workspace_id` and attach per workspace
  - Return `list[UserWorkspaceOut]`

Implementation guidance:
- Prefer SQLAlchemy `select()` and `in_()` for batching projects by workspace ids.
- Avoid N+1 queries.

- [ ] **Step 3: Wire router**

In `app/main.py`:
- import and `include_router(users_router)`
